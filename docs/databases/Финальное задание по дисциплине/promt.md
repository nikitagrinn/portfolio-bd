# Часть 1. Процесс разработки лабораторной работы с использованием LLM и анализ неточностей генерации

В рамках данного задания для разработки учебно-методического материала была выбрана документо-ориентированная NoSQL СУБД **Apache CouchDB**. 

Для получения качественного результата взаимодействие с большой языковой моделью (LLM) выстраивалось по принципу **Role-Prompting** (задание роли) и **Few-Shot Prompting** (итеративное уточнение). Ниже представлен процесс генерации, анализ типичных ошибок нейросети и способы их устранения.

## 1. Первоначальный развернутый промпт

Для того чтобы сразу ограничить нейросеть в фантазиях и задать строгий академический формат, был составлен следующий объемный промпт, учитывающий методические требования:

```text
Ты преподаватель дисциплины «Информационные системы и базы данных». Твоя задача — разработать качественную лабораторную работу для студентов педагогического направления, которые в будущем сами будут преподавать дисциплины, связанные с базами данных.

Инструмент для изучения: Apache CouchDB.
Время на выполнение: 4–8 академических часов (включая самостоятельную работу и оформление).
Лабораторная работа должна быть конкретной, шаги — воспроизводимыми, без возможности неоднозначного толкования.

В работе необходимо раскрыть ровно 3 ключевые особенности CouchDB:
1. Взаимодействие с БД исключительно через HTTP/REST API (отсутствие бинарного протокола).
2. Механизм версионирования документов (MVCC) и оптимистичные блокировки (работа с параметром _rev).
3. Концепция Schema-free и использование технологии MapReduce для фильтрации данных (написание View на JavaScript).

Сформулируй лабораторную работу в следующей структуре:
1. Название и цель работы.
2. Используемые инструменты (с указанием вариантов развертывания).
3. 4-5 последовательных задач (CRUD операции, вызов конфликта версий, создание MapReduce).
Для каждой задачи укажи: конкретные команды или скрипты, ожидаемый ответ от сервера, и что именно студент должен зафиксировать в отчет.

Запрещается использовать размытые формулировки:
- Не пиши «отправьте запрос» без указания конкретного инструмента (укажи curl).
- Не пиши «сделайте выводы» без указания того, о чем конкретно студент должен сделать вывод.
- Не предлагай установку без указания нюансов безопасности БД.

Сформулируй 4 тестовых вопроса по теории CouchDB.
Сформулируй 1 сложный вопрос для защиты (Anti-cheat), моделирующий реальную бизнес-ситуацию одновременного редактирования данных, чтобы проверить понимание механизма MVCC, а не умение гуглить.
Сформулируй шкалу оценивания из 100 баллов со штрафами за конкретные недочеты.
```

## 2. Анализ первичного ответа LLM и фиксация критических неточностей

Несмотря на подробное техническое задание, первоначальный ответ нейросети содержал ряд методических и технических уязвимостей. Из-за специфики CouchDB слепое следование инструкции от LLM привело бы к тому, что студенты не смогли бы выполнить работу.

Были зафиксированы следующие проблемы генерации:

1. **Проблема установки и настройки безопасности:** 
   * *Как ответила LLM:* «Скачайте CouchDB с официального сайта, установите и запустите».
   * *В чем ошибка:* Начиная с версии 3.0, CouchDB требует обязательной настройки пароля администратора и генерации `Erlang cookie` даже при локальной установке. Без этого база данных просто не запустится в рабочем режиме.
   * *Решение:* Нейросети было дано указание прописать два явных способа развертывания: через команду `Docker` (с передачей переменных окружения логина/пароля) и пошаговую инструкцию для Windows-установщика (с акцентом на шагах конфигурации учёток).

2. **Кроссплатформенная ошибка синтаксиса `curl` (Кавычки в Windows):**
   * *Как ответила LLM:* Для создания документа нейросеть предложила команду: `curl -X POST ... -d '{"sensor": "S1", "temperature": 25.0}'`
   * *В чем ошибка:* Использование одинарных кавычек для тела JSON отлично работает в терминалах Linux (bash) и macOS (zsh), но в стандартной командной строке Windows (`cmd.exe`) вызовет синтаксическую ошибку и запрос упадет.
   * *Решение:* Промпт был скорректирован с требованием переписать все команды с использованием двойных кавычек и экранированием внутренних элементов слэшами: `-d "{\"sensor\": \"S1\", \"temperature\": 25.0}"`. Также потребовалось добавить предупреждающую сноску для студентов.

3. **Скрытый веб-интерфейс Fauxton:**
   * *Как ответила LLM:* «Откройте веб-интерфейс и создайте MapReduce функцию».
   * *В чем ошибка:* Студент, впервые видящий CouchDB, не знает, по какому адресу находится графический интерфейс. 
   * *Решение:* Было добавлено уточнение с явным указанием пути `http://127.0.0.1:5984/_utils/` и процесса авторизации.

4. **Абстрактное воспроизведение ошибки MVCC:**
   * *Как ответила LLM:* «Обновите документ без параметра `_rev` и получите ошибку». 
   * *В чем ошибка:* Методически это слабо. Студент не понимает, как исправить ошибку.
   * *Решение:* Алгоритм был расширен. Нейросети было указано описать двухэтапный процесс: сначала получение ошибки `409 Conflict`, её фиксация для отчета, а затем успешное разрешение конфликта путем добавления актуального поля `"_rev": "<АКТУАЛЬНЫЙ_ХЭШ>"` в тело PUT-запроса.

## 3. Итеративное уточнение (Корректирующий промпт)

Для устранения выявленных галлюцинаций и недоработок, в LLM был отправлен следующий корректирующий промпт:

```text
Твой ответ хороший, но содержит технические ошибки. Исправь следующее:
1. Задание 1 (Установка): Опиши запуск через Docker одной строчкой (с флагами -e COUCHDB_USER). Для Windows опиши шаги мастера установки: создание логина/пароля администратора и выбор Random Cookie. Добавь блок "ВАЖНО", где укажешь, что во всех curl запросах нужно подставлять свои логин и пароль в URL.
2. Синтаксис curl: Перепиши все JSON-данные в командах curl. Замени внешние одинарные кавычки на двойные, а все кавычки внутри JSON экранируй слешом (\"), чтобы команды работали в стандартной cmd Windows.
3. Добавь пункт, где явно указан адрес веб-интерфейса Fauxton (/_utils/).
4. В задании с MapReduce приведи конкретный и короткий код на JavaScript, который фильтрует документы по температуре (temperature > 20), обязательно добавив проверку существования поля (if (doc.temperature)), так как БД Schema-free.
```

После применения данного корректирующего промпта был получен итоговый, технически достоверный и однозначный вариант лабораторной работы, который полностью готов к использованию в учебном процессе.
